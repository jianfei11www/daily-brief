TermiX（AACP）项目研报：AI Agent 经济清算协议

一、核心结论

  1. 项目定位明确，叙事卡位精准：TermiX 试图成为"AI Agent 世界的 DTCC（中央清算所）"，通过 AACP 协议实现 Agent 间自主商业协作、验证与结算。叙事层次高于同类 Agent 项目。
  2. 技术方案完整，但产品仍处于极早期：协议架构覆盖 TEE + zkVM 安全执行、自动化验证、去中心化评估与仲裁四大模块，白皮书已开源。但当前仅部署于 BSC 测试网，链上交易量与 Agent 数量均为零，产品未验证。
  3. 投资方 YZi Labs（原 Binance Labs）的背书尚未独立确认：官方及社媒材料中提及 YZi Labs 投资，但在公开数据库与独立信源中未检索到交叉验证。需进一步确认。
  4. 代币尚未发行，TGE 时间表不明：截至 2026 年 5 月，项目未公布代币经济模型。根据路线图，主网预计 2027 Q1 上线，代币发行可能在此之前。
  5. 综合判断：叙事层面具有 Alpha 潜力，执行层面需持续跟踪。当前阶段适合归入观察清单，不建议重仓参与。

三、项目定位与叙事

3.1 解决的问题

当前 AI Agent 在生产力层面已有突破性进展，但 Agent 之间的商业协作面临三个根本问题：

  • 信任缺失：Agent A 雇佣 Agent B 执行任务，无法确认 B 是否真正完成工作。
  • 结算障碍：Agent 之间的支付需要人工审批或中心化托管，无法实现自主清算。
  • 声誉孤岛：Agent 在一个平台积累的信誉无法移植到其他生态。

3.2 解决方案

TermiX 提出 AACP 协议，定义四个核心行业标准：

  1. 安全执行标准：TEE（Intel SGX / AMD SEV）硬件隔离 + zkVM（SP1 / RISC Zero）数学证明，确保 Agent 私钥不暴露、执行过程可验证。
  2. 自动化验证模块标准：通过 LogicHash 将商业需求转化为可编程验证逻辑，实现"交付即结算"。
  3. 去中心化评估者网络：多节点交叉验证，基于博弈论分配评估任务，不依赖单一中央裁判。
  4. 去中心化仲裁标准：链上 VRF 随机选取仲裁员，commit-reveal 投票机制，Slashing 惩罚作弊。

3.3 叙事层次

与普通 Agent 平台（如 Autonolas、Fetch.ai）不同，TermiX 的叙事站在更高维度——它不做 Agent 本身，而是做 Agent 之间的清算基础设施。类比：

  • 传统金融：Visa / Mastercard 是支付网络，DTCC 是清算所
  • Agent 经济：Autonolas / Fetch.ai 是"Agent 服务市场"，TermiX 想做"Agent 经济 DTCC"

四、技术架构

4.1 四层架构

应用层CryptoClaw — 面向用户的 Agent 执行终端，已集成近百个 Crypto Skills
协议层AACP — Job 生命周期管理、验证策略、争议解决、激励机制
执行层TEE（Intel SGX / AWS Nitro）+ zkVM（SP1 / RISC Zero）
资产层ERC-8183 Job 托管合约 + ERC-8004 Agent NFT（身份与信誉）

4.2 四种验证等级

L0人工信任模式，仅消耗 Gas
L1TEE 硬件信任，仅消耗 Gas
L2zkVM 数学信任，约 200K Gas
L3TEE + zkVM 双重验证，约 200K Gas

4.3 四种协议角色

Client（客户）发布 Job、设定预算与截止时间、以 USDC 注资托管合约
Provider（服务商）执行任务逻辑，提交交付物，获得 95% 的 Job 预算
Evaluator（评估者）运行验证脚本，判定交付物是否合格，获得 3–4% 评估费用
Arbitrator（仲裁员）争议时的最终裁决，通过 commit-reveal 投票，获得争议押金分成

4.4 质押经济模型

所有参与者维持一个持久 USDC 质押池：

  • Client：Job 预算 × 5% 锁定
  • Provider：Job 预算 × 10%（可因高验证等级获得最高 30% 折扣）
  • Evaluator：Job 预算 × 10%
  • Arbitrator：固定 10 USDC / 每次争议
  • 声誉系数（0–100）降低锁定：声誉越高，所需质押越少
  • 作弊惩罚：Slashing 从锁定余额扣除，初犯罚 60%，三犯及以上罚 100%

七、风险评估

7.1 高风险（🔴）

  1. YZi Labs 投资未独立验证：项目宣称 YZi Labs（原 Binance Labs）投资，但在公开投资公告数据库、YZi Labs 官网及独立信源中未检索到交叉确认。此为关键前提，需优先核实。
  2. 产品处于原型阶段：app.termix.ai 显示总交易量 $0、Agent 注册数 0、Job 数 0。协议架构虽完整，但无实际使用数据验证。
  3. 团队背景不透明：未在官网、白皮书、GitHub 中找到核心团队成员的实名信息、过往经历与行业背景。
  4. 叙事依赖性：项目的全部价值建立于"AI Agent 经济"这一尚未被市场大规模验证的叙事之上。若赛道遇冷，项目将面临叙事溢价归零风险。

7.2 中等风险（🟡）

  1. 竞品密集，先发劣势：Autonolas、Fetch.ai 已拥有代币流动性与社区生态，TermiX 起步明显晚于对手。
  2. 执行周期长：路线图跨越至 2027 Q1，期间团队执行力、资金储备、技术难度均为变量。
  3. 代币经济模型未定：无代币即无估值锚点，无法进行投资回报测算。
