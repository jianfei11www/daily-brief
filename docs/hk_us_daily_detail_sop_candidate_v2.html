港美股日报细节候选SOP v2

港美股日报首页细节锁定 SOP v2.0 候选稿

Status: candidate, not yet permanent.

本文件用于锁定本轮已讨论确认的港美股日报首页细节，重点防止后续生成、开发、测试或发布时再次漂移。它比 v1 更细，目标是做到：照着本文件即可生成接近成品的首页结构，并能明确判断哪里不合格。

在用户确认前，本文件不替代以下正式文件：

  • `docs/hk_us_daily_sop_v1.md`
  • `docs/hk_us_daily_development_principles.md`
  • `docs/hk_us_daily_development_sop_v1.md`
  • `docs/hk_us_daily_test_plan_v1.md`

确认后，应把本文件拆分合并进正式四份文档，并同步更新测试。

1. 本 SOP 锁定范围

本 SOP 只锁定港美股日报首页的细节呈现规则，特别是：

1. `今天决策摘要`

2. `1. 港股打新`

3. 港股打新子页面入口

4. ETF / 基金产品是否展示

5. 数据说明和信源说明的位置

6. 首页与子页面的信息边界

7. 发布前的人工检查和自动化测试断言

本 SOP 不重新定义全部港美股日报数据源、不重新定义所有采集逻辑、不替代完整内容 SOP。

2. 首页阅读路径锁定

首页必须让读者按以下顺序阅读：

1. 先看 `今天决策摘要`，判断今天总风险环境和行动方向。

2. 再看 `1. 港股打新`，只知道当前哪些项目值得进一步进入子页面。

3. 再看 `2. 美股重大事件`，理解美股主线和风险来源。

4. 再看 `3. 重点新股上市及重大科技融资`，判断是否有新增资本市场机会。

5. 再看 `4. 美股指数`、`5. 全球经济指标`、`6. A股 & 港股`，验证前文判断是否有数据支撑。

6. 如港股打新值得研究，再点击子页面看项目级细节。

首页不能让读者在第一屏就被港股打新细节、ETF 空模块、数据说明或重复结论打断。

3. `今天决策摘要` 的职责

3.1 唯一定位

`今天决策摘要` 是首页的全局投资判断区，不是新闻摘要，不是数据摘要，不是港股打新摘要。

它只回答四个问题：

1. 今天整体风险偏好如何。

2. 今天最重要的宏观确认项是什么。

3. 港股打新在今天的阅读顺序和决策位置是什么。

4. 今天有没有足够确定的新股、融资或重大事件可以提高操作优先级。

3.2 必须输出的四类结论

正常情况下，`今天决策摘要` 使用 3-4 条结论。优先使用 4 条：

1. 总仓位 / 风险偏好结论。

2. 宏观压力 / 确认变量结论。

3. 港股打新阅读顺序结论。

4. 新股 / 融资 / 重大机会准入结论。

如果当天市场信息很少，最低保留 3 条：

1. 总风险结论。

2. 最重要确认变量。

3. 今日动作。

不得少于 3 条；不得超过 4 条。超过 4 条会削弱首页判断密度。

3.3 每条结论的固定句式

每条结论使用以下结构：

```text

强结论短句。依据和动作说明。

```

强结论短句要求：

  • 8-18 个汉字左右。
  • 句末用句号。
  • 可加粗。
  • 必须能单独读懂。
  • 必须是判断，不是标题。

依据和动作说明要求：

  • 1 句为主，最多 2 个分句。
  • 必须说明依据来自宏观、指数、风险偏好、事件或准入规则。
  • 必须给出行动含义，例如防守、等待、避免追高、进入子页面、降低优先级。

合格示例：

```text

短线防守优先。风险偏好偏弱、宏观压力高，美股趋势和科技情绪同步偏弱；当前不适合追高成长股和港股高弹性方向。

```

不合格示例：

```text

市场需要继续观察。

```

不合格原因：没有说明观察什么、为什么观察、动作是什么。

3.4 四条结论的标准模板

#### 第 1 条：总风险结论

职责：告诉读者今天应偏防守、偏进攻还是等待。

输入字段：

  • `market_state.risk_appetite`
  • `market_state.macro_pressure`
  • `market_state.equity_trend`
  • `market_state.tech_sentiment`
  • `market_state.hk_sentiment`

推荐句式：

```text

短线防守优先。风险偏好{状态}、宏观压力{状态}，美股趋势和科技情绪{状态}；当前{动作建议}。

```

动作建议可选：

  • `不适合追高成长股和港股高弹性方向`
  • `适合先观察指数修复，不急于提高仓位`
  • `可小幅提高风险资产观察优先级，但仍需确认宏观变量`

禁止：

  • 只列 `风险偏好 偏弱 / 宏观压力 高 / 美股趋势 偏弱`。
  • 把状态拆成表格或标签。
  • 写成“总评：风险偏好偏弱；宏观压力高；……”这种机械拼接。

#### 第 2 条：宏观确认项

职责：告诉读者今天最重要的确认变量是什么。

输入字段：

  • `macro_indicators` 中的 VIX、10Y 美债、美元指数、原油、黄金。
  • `macro_summary`
  • `market_state.key_risks`

推荐句式：

```text

宏观变量是今天最重要的确认项。{变量1}、{变量2}、{变量3}仍在{压制/支撑}风险资产；只有{反向条件}，压力判断才可下调。

```

合格变量：

  • VIX
  • 10Y 美债
  • 美元指数
  • 原油
  • 黄金
  • Fed 预期
  • 通胀数据

禁止：

  • 写“关注宏观变化”但不列变量。
  • 写“宏观风险较高”但不给反向条件。
  • 写结论和后文宏观表格相反。

#### 第 3 条：港股打新阅读顺序

职责：只说明港股打新在首页的决策位置，不展开项目细节。

推荐句式：

```text

港股打新不在摘要里抢结论。今天先完成美股、宏观和港股科技情绪判断，再到港股打新板块看本轮项目优先级；具体项目排序和热度不在总摘要里展开。

```

允许替代表达：

```text

港股打新先看环境再看项目。先判断美股科技情绪、恒生科技和流动性环境，再进入第 1 部分和子页面看项目级申购判断。

```

禁止：

  • 在这条里出现具体项目名。
  • 在这条里出现 `UISEE`、`上海拓数控`、`Robot Phoenix`、`TenNor`。
  • 在这条里出现 `券商孖展`、`入场费`、`一手中签率`、`暗盘`、`基石投资者`。
  • 写“UISEE 是优先研究对象”。
  • 写“上海拓数控和 Robot Phoenix 属于高热度博弈观察”。

这些内容只能进入 `1. 港股打新` 或港股打新子页面。

#### 第 4 条：新股 / 融资 / 事件准入

职责：说明是否存在足够确定的新股、融资、IPO、重大科技事件可以进入核心判断。

推荐句式：

```text

新股和融资机会不强行升级。本窗口没有足够高确定性的美股 IPO 或重大科技融资线索；低置信内容不进入核心判断。

```

如果当天有强事件：

```text

宏观事件优先级高于新股线索。{事件}已由{来源}确认，短线影响{行业/指数/公司}；但仍需观察{后续变量}。

```

禁止：

  • 把低置信 ETF 传闻写入核心结论。
  • 把普通融资新闻写成机会。
  • 把未经官方确认的产品上线写成确定事实。

3.5 摘要区域禁止词和禁止字段

`今天决策摘要` 区域内禁止出现以下词或字段，除非用户当日明确要求：

```text

UISEE

上海拓数控

Robot Phoenix

TenNor

券商孖展

入场费

一手中签率

暗盘热度

基石投资者

项目判断表

细节入口

来源：

置信度：

ETF / 基金产品

本窗口无合格项目

数据说明：

模板版本

时间窗口

数据时间戳

市场会话

```

说明：

  • 摘要区不是信源区，所以不放 `来源：`。
  • 摘要区不是打新区，所以不放项目名。
  • 摘要区不是数据区，所以不放 `数据说明：`。

3.6 摘要区长度

PC 页面：

  • 每条 1-2 行。
  • 4 条合计不超过 380 个中文字符。

移动端：

  • 每条可以自然换行。
  • 不得出现长表格、长 URL、来源列表。

超长时的处理优先级：

1. 删除项目名。

2. 删除重复状态词。

3. 删除来源。

4. 删除解释性废话。

5. 保留动作建议和反向变量。

3.7 摘要必须按「因果链」组织（强制）

摘要不是散点罗列，必须按投资者可理解的因果链写：

```text

原因（宏观约束/政策/地缘）→ 盘面验证（指数/波动率/汇率/商品）→ 今日动作（仓位与纪律）→ 反向触发（什么条件下切换策略）

```

强制规则：

  • 先写原因，再写盘面；不得先“短线防守/进攻”再补原因。
  • 盘面必须引用页面已有的“收盘数据/口径”或已确认的事件，不得凭感觉写“走弱/走强”。
  • 动作必须可执行：控制追高、降低高β暴露、等待确认、仅做低仓位试探等；禁止“继续观察”。
  • 反向触发必须写成可验证条件（见 3.9 固定阈值），避免“如果压力缓和”这种空话。

验收问题（任何一条答不上来即不合格）：

1. 为什么今天要防守/进攻？

2. 今天明确要做什么/不做什么？

3. 满足什么条件可以改变动作？

3.8 指标命名与简写规则（中文优先，兼顾专业读者）

目标：既不全英文吓退不熟悉美股的人，又不把版面写成教科书。

规则：

1. 首次出现：使用「中文简称 +（英文缩写/代码，可选）+ 一句中文解释」。

2. 后续出现：只使用中文简称（必要时保留缩写），禁止在同一句反复堆全称。

3. 解释句只出现一次：放在首次出现的同条结论/同段判断里，后续不重复。

推荐写法（首次出现示例）：

  • `10年美债收益率（10年美债，衡量美国长期利率水平）`
  • `VIX恐慌指数（VIX，衡量市场波动与避险情绪）`
  • `布伦特原油（Brent，衡量全球油价与能源风险溢价）`
  • `纳斯达克100指数（纳指100/NDX，反映美股科技权重走势）`
  • `纳指100ETF（QQQ，纳指100交易盘代表）`

后续写法：

  • `10年美债`、`VIX` / `VIX恐慌指数`、`布伦特原油`、`纳指100`、`QQQ`

禁止写法：

  • 同一条结论里同时出现 `美国10年期国债收益率（10年美债）` + `10Y 美债` 反复切换口径。
  • 全段只写 `10Y`、`DXY`、`Brent` 等英文缩写不给中文解释。

3.8.1 表格行必须中文化（强制）

`4. 美股指数` 与 `5. 全球经济指标` 的表格行名必须以中文为主；若保留英文/代码，只能放在括号里作为补充。

合格示例：

  • `标普500（S&P 500）`
  • `纳指100（NDX）`
  • `道指（道琼斯）`
  • `10年美债收益率（10年美债）`
  • `高收益债ETF（HYG）`

不合格示例：

  • 只写 `S&P 500`、`Nasdaq Composite`、`HYG` 而没有中文说明。

3.9 固定阈值与收盘口径（强制）

本日报面向公开读者，所有“偏高/偏强/偏紧/转弱”等判断必须尽量落到可验证阈值，避免模糊话。

口径锁定：

  • 所有阈值判断默认使用“最近一个交易日收盘值/收盘收益率”（不使用盘中）。
  • 若为周末/休市日版本，必须在页面元信息或数据说明中写明“数据截至最近交易日收盘”。

固定阈值（沿用主流市场共识，后续如需调整必须显式改版本）：

  • `10年美债`：收盘 **≥ 4.5%** 视为“利率高压线”；收盘 **≤ 4.3%** 视为“缓和线”。
  • `VIX`：收盘 **≥ 20** 视为“波动偏高线”；收盘 **≤ 18** 视为“缓和线”。
  • `布伦特原油`：收盘 **≥ 90 美元/桶** 视为“油价偏强线”；收盘 **≤ 85 美元/桶** 视为“缓和线”。

写法要求：

  • 不允许只写“利率偏紧/波动率偏高/油价偏强”而不写阈值或收盘数值。
  • 若页面已有收盘数值（例如表格里已给出），摘要中应优先引用该数值；否则只能写阈值触发条件，不得编造具体数值。
  • 阈值表达必须是“方向型参考”，禁止绝对化口吻（例如“写死/必要条件/必须/需回到”）。

3.10 公开版口吻禁用词（强制）

本日报为公开版简报，禁止出现内部讨论/审稿待办口吻与“绝对化承诺”口吻。

#### 3.10.1 禁用词（出现即判定不合格，必须重写）

  • `写死`、`必要条件`、`必须`（用于给读者下死命令的语气）、`需回到`（强制结论）
  • “需补充…后再更新”“暂不能仅凭…形成完整主线”等内部审稿待办句式

#### 3.10.2 推荐替换范式（方向型参考，不做绝对承诺）

  • `写死：…为必要条件` → `参考：…可作为优先观察条件之一`
  • `需回到防守框架` → `可考虑回到防守框架/偏向防守框架`
  • `必须…` → `建议…/优先…/倾向…`

4. `1. 港股打新` 首页板块职责

4.1 唯一定位

`1. 港股打新` 是首页里的打新入口判断，不是完整打新报告。

它只回答三个问题：

1. 当前港股打新整体是否值得点进子页面。

2. 哪些项目是当前优先研究、博弈观察或暂不主打。

3. 去哪里看完整申购细节。

4.2 固定结构（已更新：首页不展示“综合评述/综合判断”标签）

首页必须使用以下结构：

```markdown

1. 港股打新

{一句话项目优先级判断（不加“综合评述/综合判断”标签）}

细节入口：private/hk-us-ipo/YYYYMMDD-hk-ipo.html

来源：{source_names}｜置信度：{高/中/低}

```

固定项解释：

  • `## 1. 港股打新`：必须存在，不能删除。
  • 首页不再展示 `### 综合评述` 和 `综合判断：` 标签（避免短内容重复占版面）。
  • 判断句必须保留且只出现一段。
  • `细节入口：`：必须存在，且 HTML 必须可点击。
  • `来源：...｜置信度：...`：必须存在，但视觉上可以弱化。

4.3 综合判断的固定结构

综合判断建议使用以下结构：

```text

当前港股打新{总策略}：{优先研究项目}；{博弈观察项目}；{暂不主打项目}。

```

标准示例：

```text

当前港股打新只选强热度、强主题项目：UISEE 是优先研究对象；上海拓数控和 Robot Phoenix 属于高热度博弈观察；TenNor 热度较弱，暂不作为主打申购方向。

```

如果当天无合格项目：

```text

当前港股打新不强行参与：本窗口没有同时满足强主题、强热度和可验证申购数据的项目，主页面只保留入口，等待子页面补齐券商热度和暗盘变量。

```

如果当天只有一个项目：

```text

当前港股打新只保留单项目研究：{项目名} 符合{主题/热度/数据}条件，适合进入子页面复核；其余披露项目暂不进入主线判断。

```

4.4 综合判断必须包含的投资信息

一段综合判断里至少包含以下 2 类信息：

1. 参与态度：优先研究、博弈观察、小仓测试、暂不主打、不强行参与。

2. 项目排序：至少指出谁更值得看、谁只观察、谁暂缓。

如果没有项目排序，就不是合格综合判断。

允许的判断词：

  • `优先研究对象`
  • `高热度博弈观察`
  • `小仓测试`
  • `暂不作为主打申购方向`
  • `暂不强行参与`
  • `等待暗盘和券商热度确认`
  • `只适合子页面复核`

禁止的空泛词：

  • `值得关注`
  • `继续跟踪`
  • `有跟踪价值`
  • `需要结合市场判断`
  • `后续观察`
  • `主页面不展开`

这些词如果单独出现，没有项目排序和动作含义，视为不合格。

4.5 首页港股打新禁止内容

以下内容禁止出现在首页 `1. 港股打新`：

```text

今日重点：

风险变量：

优先关注：

博弈观察：

暂不主打：

券商孖展：
```

2. 子页面字段说明。

6. ETF / 基金产品规则

6.1 默认删除

首页默认不展示 `ETF / 基金产品`。

禁止出现：

```text

ETF / 基金产品

本窗口无合格项目。

ETF / 基金产品

Cerebras ETF 产品观察

```

6.2 可进入首页的例外

ETF 只有同时满足以下条件，才允许进入首页相关正文：

1. 有 SEC、交易所、发行方或公司 IR 等官方确认。

2. 事件影响美股市场、主题产品供给、监管审批或上市公司股价。

3. 不是单纯产品列表或二手传闻。

4. 能写出市场影响、风险提示、后续变量。

6.3 例外放置位置

即使满足例外，也不得恢复 `ETF / 基金产品` 空模块。

放置规则：

  • 如果是重大市场事件，放入 `2. 美股重大事件`。
  • 如果是 IPO / 资本市场产品且高度相关，放入 `3. 重点新股上市及重大科技融资` 的合适子项。
  • 如果只是低置信线索，不进入正文，只进入待补或内部日志。

6.4 `2. 美股重大事件`（宏观大事件）准入与写法（强制）

本板块只允许“新闻事件本体”进入，禁止把市场结果/行情归因当作事件。

#### 6.4.1 什么能进入（准入条件）

只允许以下类型进入（至少满足其一）：

1. 地缘政治/战争/停火/制裁/能源通道：会改变能源、供应链、航运与风险溢价。

2. 货币政策/财政政策/监管：会改变利率预期、流动性与风险偏好。

3. 关键宏观数据：会改变增长/通胀预期与政策路径（例如 CPI、非农等）。

4. 具有制度性影响的金融事件：会改变市场结构或风险定价方式（例如重大监管执法、系统性风险处置）。

明确禁止：

  • `油价上行/收益率上行/指数回撤` 这类**市场结果**作为“事件”。
  • `因为油价上涨所以美股回撤` 这种**行情归因**放在“事件”行。

正确做法：

  • 事件行写“世界发生了什么”（谁做了什么/发生了什么）。
  • 判断行再写“通过油价/利率/VIX如何传导到风险偏好与操作动作”（可引用 3.9 阈值）。

#### 6.4.2 事件行必须满足「三要素」且精炼

事件行一条必须满足：

```text

谁/什么主体 + 做了什么/发生了什么 + 关键对象/地点/机制

```

示例（结构示意，不代表真实内容）：

  • `事件：伊朗/以色列冲突风险升级，相关方释放进一步军事行动信号。`
  • `事件：美联储官员释放更偏鹰派/鸽派的政策信号，利率路径预期发生变化。`
  • `事件：美国关键通胀/就业数据超预期，市场重新定价降息节奏。`

#### 6.4.3 判断行必须给出传导链 + 动作 + 反向触发

判断行必须覆盖：

1. 传导链：这件事如何影响 `布伦特原油/10年美债/VIX/美元` 等关键变量。

2. 今日动作：读者今天该做什么/不做什么。

3. 反向触发：什么条件下可以改变动作（优先用 3.9 固定阈值）。

#### 6.4.4 自动化筛选两套方案（试运行口径）

为了避免“地缘一件事占满整版”或“全是政治八卦”，宏观事件候选必须由系统自动抓取并筛选，输出两套结果供试运行对比：

  • 方案 A：`balanced_v1`（均衡型）
  • 目标：地缘/政策/宏观数据三类各取 1 条为底座，最多 4 条。
  • 适用：需要每日覆盖更全面，不被单一主题绑架。
  • 方案 B：`top_impact_v1`（影响力优先）
  • 目标：按评分取前若干条，每类最多 2 条，至少覆盖 2 个类别。
  • 适用：当天确实有强事件时，优先把“最影响市场的”放到前面。

两套方案都必须满足：

  • 事件必须是“新闻事件本体”，并通过“宏观相关性过滤”（不收纯民生、纯党争、纯八卦）。
  • 如果官方/主流媒体源不可访问，允许输出空集，但必须在内部产物里记录抓取错误；不得用旧新闻凑数。

#### 6.4.5 宏观事件标题必须中文可读（强制）

宏观大事件面向中文读者，标题必须以中文主题表达为主。若信源标题为英文，只能作为括号内补充，且必须做“主题中文化”处理（例如：`伊朗相关进展（原英文标题…）`）。

不合格示例：

  • 直接把 RSS 英文标题原样当作标题，导致整节基本是英文列表。

7. 数据说明和来源展示

7.1 数据说明定义

数据说明包括：

```text

数据说明：截至 2026-05-15 美股收盘；来源 China.org / MarketScreener / AP 交叉校验。

```

它是口径说明，不是投资结论。

7.2 放置位置

数据说明必须放在对应板块末尾。

推荐顺序：

1. 表格。

2. 简要评述。

3. 数据说明。

禁止顺序：

1. 数据说明。

2. 表格。

3. 简要评述。

原因：读者应先看数据和判断，再看口径说明。

7.3 样式要求

HTML 中数据说明应使用弱化样式：

```html

<p class="source-note">数据说明：...</p>

```

视觉要求：

  • 字号小于正文。
  • 颜色弱于正文。
  • 不使用卡片强调。
  • 不使用大段背景块。
  • 不进入今天决策摘要。

7.4 来源与置信度

来源与置信度在分析型条目中必须保留，但不能抢版面。

港股打新首页来源行：

```text

来源：HKEX、uSMART、The Standard、IPOX、Infocast｜置信度：高

```

不允许写成：

```text

数据源说明：我们查阅了 HKEX、uSMART、The Standard、IPOX、Infocast，综合判断置信度较高……

```

原因：太长，会把页面变成解释文档。

9. 生成步骤

9.1 生成前检查

生成首页前，必须确认：

1. 本次是港美股日报，不是区块链日报。

2. 区块链内容没有混入，除非有港美股资本市场关联。

3. 港股打新有 `overall_judgment`。

4. 港股打新有 `detail_url`。

5. 港股打新子页面文件存在或将同步生成。

6. 美股指数、全球宏观、A/H 指数有数据时间或数据说明。

7. 低置信 ETF 线索已排除。

9.2 生成 `今天决策摘要`

步骤：

1. 读取 `market_state`。

2. 形成总风险结论。

3. 读取 `macro_indicators` 和 `macro_summary`。

4. 形成宏观确认项。

5. 写港股打新阅读顺序，但不读取项目明细。

6. 检查 IPO / 融资 / 美股重大事件是否有高确定性机会。

7. 输出 3-4 条卡片。

8. 扫描摘要区禁止词。

禁止在第 5 步读取：

  • `hk_ipo_summary.projects`
  • `hk_ipo_summary.detail_fields`
  • `hk_ipo_summary.risk_variables`
  • `hk_ipo_summary.detail_notes`

9.3 生成 `1. 港股打新`

步骤：

1. 读取 `hk_ipo_summary.overall_judgment`。

2. 读取 `hk_ipo_summary.detail_url`。

3. 读取 `hk_ipo_summary.sources`。

4. 读取 `hk_ipo_summary.confidence`。

5. 输出固定三段。

6. 扫描首页港股打新禁止词。

7. 确认子页面链接可点击。

禁止在首页输出：

  • `projects`
  • `detail_fields`
  • `detail_notes`
  • `risk_variables`

9.4 生成子页面

步骤：

1. 读取 `projects`。

2. 读取 `detail_fields`。

3. 读取 `risk_variables`。

4. 生成项目判断表。

5. 生成关键字段表。

6. 生成阶段说明。

7. 添加返回首页链接。

8. 检查首页点名项目是否都在子页面存在。

10. 人工验收标准

人工验收时，打开首页只看前两部分，按以下问题判断：

10.1 今天决策摘要验收

7. 摘要是否没有数据说明。

12. 失败处理

12.1 摘要失败

如果摘要出现项目名：

处理：

1. 删除摘要中的项目名。

2. 改写为“进入港股打新板块看项目优先级”。

3. 保留项目判断到第 1 部分或子页面。

4. 补回归测试。

12.2 港股打新首页失败

如果首页港股打新太长：

处理：

1. 保留综合判断第一段。

2. 保留细节入口。

3. 保留来源/置信度。

4. 其余移动到子页面。

不得简单删除项目判断。首页仍要能看出项目优先级。

12.3 ETF 失败

如果首页出现 ETF 空模块：

处理：

1. 删除 ETF 子模块。

2. 检查渲染器是否仍循环 `funds` 或 `ETF / 基金产品`。

3. 补测试 `assertNotIn("ETF / 基金产品", output)`。

12.4 数据说明失败

如果数据说明占版面：

处理：

1. 移到对应板块末尾。

2. 使用 `source-note` 样式。

3. 不改变数据本身。
