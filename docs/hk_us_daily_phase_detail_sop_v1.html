0. 傻瓜执行总流程

执行者不需要自行设计流程，必须按以下顺序逐步执行。

```text

第 1 步：确认日期和输出文件

第 2 步：收集宏观大事件和产业热点候选

第 3 步：筛选高影响事件

第 4 步：写今天决策摘要

第 5 步：写港股打新首页

第 6 步：写宏观大事件

第 7 步：写产业热点与科技主线

第 8 步：生成页面

第 9 步：按检查清单逐项验收

第 10 步：运行测试和质量门禁

第 11 步：按固定版本流程推送

第 12 步：用版本 URL 验证

```

任何一步不通过，不得跳到后一步。尤其是第 9-12 步，不允许因为“看起来差不多”而省略。

如果当天没有对应文件，先创建或补齐原始数据，不得直接改最终 HTML 当成正式生产。

```text
outputs/YYYY-MM-DD-stage-hk-us-daily.html
```

不得随意改文件名。文件名改动会导致后续发布、日志和验证断链。

```text

```

线上 URL 固定为：


1. 总目标

港美股日报不是新闻罗列，也不是研究报告长文。它必须是一份高度压缩的投资决策简报。

每个板块必须回答：

  • 今天发生了什么。
  • 对风险偏好、资产价格或投资方向有什么影响。
  • 接下来需要看什么变量。
  • 结论的来源和置信度是什么。

不得出现：

  • 大段背景铺陈。
  • 口语化聊天表达。
  • 重复解释。
  • 没有动作含义的废话。
  • 只有信息、没有判断。
  • 只有判断、没有来源。

2. 全局语言标准

2.1 语气必须正式

日报面向大众阅读，正文必须使用正式、规范、可公开发布的表达。

禁止写法：

```text

宏观变量是今天最重要的确认项。

这个需要先看看。

这个事情很重要。

现在不要抢结论。

```

推荐写法：

```text

宏观压力仍需验证。

油价、10Y 美债、VIX 与美元指数仍对风险资产形成约束；若 VIX 回落且 10Y 美债下行，短线压力可下调。

```

2.2 每句话必须有信息密度

每句话必须至少承担一种功能：

  • 给出结论。
  • 说明依据。
  • 给出交易变量。
  • 说明反向变量。
  • 标注来源和置信度。

如果一句话删掉后不影响投资判断，就应该删除。

必须删除的废话类型：

```text

新股和融资机会不强行升级。

当前需要继续观察。

后续再看情况。

需要结合市场判断。

主页面只适合做综合判断。

```

这些表达只有在后面紧跟明确变量、动作或来源时才允许保留。

3. `今天决策摘要` SOP

3.1 职责

`今天决策摘要` 只负责全局投资判断，不负责展开港股打新、不负责讲项目排序、不负责重复后文事件。

它回答三个问题：

1. 今天整体风险偏好如何。

2. 今天最重要的宏观或市场确认项是什么。

3. 今天美股/宏观/科技主线覆盖是否足够形成判断。

3.2 输出数量

固定输出 3-4 条。

默认 3 条：

1. 总风险判断。

2. 宏观压力判断。

3. 主线覆盖判断。

只有在确实存在高质量新增主线时，才允许第 4 条。

禁止：

  • 输出状态表。
  • 输出 `今日动作 / 关键风险 / 后续观察` 长列表。
  • 在摘要里讲港股打新具体项目。
  • 使用“港股打新不在摘要里抢结论”这类自我解释句。

3.3 标准句式

每条使用：

```text

强结论短句。依据 + 动作含义。

```

合格示例：

```text

短线防守优先。风险偏好偏弱、宏观压力高，美股趋势和科技情绪同步偏弱；当前不适合追高成长股和港股高弹性方向。

```

```text

宏观压力仍需验证。油价、10Y 美债、VIX 与美元指数仍对风险资产形成约束；若 VIX 回落且 10Y 美债下行，短线压力可下调。

```

```text

宏观主线覆盖改善。本轮已纳入油价与利率扰动、美伊冲突升级风险及谈判条件变化；短线重点观察油价、VIX、10Y 美债与 Nasdaq 的联动。

```

不合格示例：

```text

宏观变量是今天最重要的确认项。

```

不合格原因：口语化、没有动作含义、没有说明变量如何影响判断。

```text

港股打新不在摘要里抢结论。今天先完成美股、宏观和港股科技情绪判断，再到港股打新板块看本轮项目优先级；具体项目排序和热度不放在总摘要里展开。

```

不合格原因：摘要下方马上就是港股打新，提前解释页面结构会打断阅读。

3.4 摘要禁用内容

`今天决策摘要` 禁止出现：

  • 港股打新具体项目名。
  • 港股打新排序。
  • 子页面说明。
  • 数据来源。
  • 置信度。
  • 大段后续观察。
  • 模板解释。
  • “不强行升级”“不抢结论”等自我说明。

3.5 傻瓜操作步骤

写 `今天决策摘要` 时按下面顺序执行。

#### 第 1 步：读取市场状态

必须先看以下字段：

```text

market_state.risk_appetite

market_state.macro_pressure

market_state.equity_trend

market_state.tech_sentiment

market_state.hk_sentiment

```

如果这些字段缺失，先运行分析流程，不得凭感觉写摘要。

#### 第 2 步：写第 1 条

模板：

```text

短线防守优先。{风险偏好 + 宏观压力 + 美股/科技情绪}；{今日动作}。

```

今日动作只能写：

  • 控制追高。
  • 降低高弹性方向优先级。
  • 等待关键变量确认。
  • 谨慎参与。

不得写：

  • 继续观察。
  • 看情况。
  • 不抢结论。

#### 第 3 步：写第 2 条

模板：

```text

宏观压力仍需验证。{油价/10Y美债/VIX/美元指数的状态}；若{反向变量}，{判断如何调整}。

```

必须至少出现两个宏观变量，例如：

```text

油价、10Y 美债、VIX、美元指数

```

不得写：

```text

宏观变量是今天最重要的确认项。

```

#### 第 4 步：写第 3 条

模板：

```text

宏观主线覆盖改善。{已纳入的高影响事件类型}；短线重点观察{3-4 个变量}。

```

如果宏观大事件少于 2 条，不得写“覆盖改善”。改写为：

```text

宏观主线仍需补强。当前高影响事件覆盖不足；发布前需继续补充 Reuters/AP/CNBC/Bloomberg、金十、财联社及官方来源。

```

#### 第 5 步：决定是否写第 4 条

只有满足以下任一条件才写第 4 条：

  • 产业热点出现高置信新主线。
  • 宏观大事件出现 3 条以上高影响事件。
  • 港美股有明确交易影响的监管、财报、政策或产业链事件。

否则保持 3 条，不得为了凑数写废话。

#### 第 6 步：删除摘要禁用内容

写完后逐项搜索：

```text

新股和融资机会不强行升级

港股打新不在摘要里抢结论

宏观变量是今天最重要的确认项

今日动作：

关键风险：

后续观察：

```

任何一个出现，必须删除或重写。

4. `1. 港股打新` SOP

4.1 首页职责

港股打新首页只保留：

1. 综合评述。

2. 子页面入口。

3. 来源与置信度。

不得在首页展开：

  • 全量项目表。
  • 今日重点多条卡片。
  • 风险变量长段。
  • 项目级数据。
  • 孖展、入场费、一手中签率、暗盘、基石投资者详细表。

这些全部放入港股打新子页面。

4.2 首页综合评述标准

综合评述必须能直接指导下一步动作。

合格方向：

```text

综合判断：当前港股打新只选强热度、强主题项目：UISEE 是优先研究对象；上海拓数控和 Robot Phoenix 属于高热度博弈观察；TenNor 热度较弱，暂不作为主打申购方向。

```

不合格方向：

```text

综合判断：HKEX 披露项目仍有跟踪价值，主页面不展开全量名单。

```

不合格原因：没有告诉读者优先看谁、为什么、如何行动。

4.3 标题规则

页面已经有 `1. 港股打新`，板块内小标题只写：

```text

综合评述

```

不得写：

```text

港股打新综合评述

```

原因：重复标题，降低版面密度。

4.4 子页面入口

入口必须可点击，格式：

```text

细节入口：private/hk-us-ipo/YYYYMMDD-hk-ipo.html

```

发布后必须验证链接可打开。

4.5 傻瓜操作步骤

写 `1. 港股打新` 时只做三件事。

#### 第 1 步：写综合评述

格式固定：

```text

综合判断：{当前是否参与}；{优先研究对象}；{博弈观察对象}；{暂不主打对象}。

```

必须回答：

  • 谁优先研究。
  • 谁只能博弈观察。
  • 谁暂不主打。
  • 为什么。

不得只写：

```text

可进入子页面跟踪。

仍有研究价值。

需要结合宏观判断。

```

#### 第 2 步：写细节入口

格式固定：

```text

细节入口：private/hk-us-ipo/YYYYMMDD-hk-ipo.html

```

不得写成纯文本后无法点击。HTML 中必须渲染为链接。

#### 第 3 步：写来源与置信度

格式固定：

```text

来源：HKEX、uSMART、The Standard、IPOX、Infocast｜置信度：高

```

如果缺少 HKEX，置信度不得为高。

#### 第 4 步：删除多余内容

首页港股打新部分不得出现：

```text

今日重点：

风险变量：

优先关注：

博弈观察：

暂不主打：

项目判断表

券商孖展

一手中签率

暗盘

基石投资者

```

这些内容只能放到子页面。

5. `2. 宏观大事件` SOP

5.1 命名规则

第 2 部分必须命名为：

```text

2. 宏观大事件

```

不得再使用：

```text

2. 美股重大事件

```

原因：该板块已经覆盖油价、利率、VIX、美元、战争、停火、地缘政治等跨市场变量，不只影响美股。

5.2 入选范围

可入选：

  • Fed、CPI、PCE、非农、美债、美元、VIX、黄金、原油。
  • 战争、停火、制裁、航运、能源供给。
  • 影响全球风险偏好的重大政策和监管事件。
  • 会映射到美股、港股、A/H 科技资产或风险资产的全球事件。

不得入选：

  • 只有标题、没有市场影响的快讯。
  • 普通 crypto 二级市场涨跌。
  • 单纯个股涨跌。
  • 小公司低影响新闻。
  • 无来源确认的传闻。

5.3 固定格式

每条宏观大事件固定三段式：

```text

标题

事件：一句话说明发生了什么。

判断：一句话说明投资含义，可追加一句反向变量。

来源：来源 A、来源 B｜置信度：高/中/低

```

不得使用旧格式：

```text

事件说明：...

市场判断：...

影响范围：...

风险提示：...

后续变量：...

```

5.4 三段式写法标准

事件：

  • 只写事实。
  • 一句话。
  • 不写推论。
  • 不写背景长文。

判断：

  • 先写影响方向。
  • 写清影响对象。
  • 写清反向变量。
  • 一般 1-2 句。

来源：

  • 必须放在条目末尾。
  • 必须有置信度。
  • 快讯源不能单独支撑高置信结论。

合格示例：

```text

伊朗警告美国若再发起军事行动将面临进攻性回应

事件：伊朗警告美国若再军事行动，将面临进攻性回应。

判断：美伊风险升级会推升油价和避险需求，压制科技股风险偏好。若谈判恢复且油价回落，冲击减弱。

来源：金十数据、Xinhua｜置信度：高

```

不合格示例：

```text

事件说明：2026-05-17 金十数据重要事件与新华社均显示，伊朗军方称若美国再次发起军事行动，美国在地区内的军事资产及部队将面临新的进攻性回应。

市场判断：该事件是油价、VIX 和美股风险偏好的高优先级变量：若军事行动升级，能源供给风险和避险需求可能上升，科技股和高估值资产容易受到风险溢价上行压制。

```

不合格原因：过长、标签过多、阅读成本高。

5.5 傻瓜操作步骤

写 `2. 宏观大事件` 时按下面步骤执行。

#### 第 1 步：先建候选清单

候选必须来自以下来源池：

```text

金十数据重要事件

财联社电报

Reuters

AP

CNBC

Bloomberg

Xinhua

National Business Daily

官方部门/监管/央行/交易所

```

每天至少检查：

  • 地缘政治。
  • 油价。
  • 10Y 美债。
  • VIX。
  • 美元指数。
  • Fed / CPI / PCE / 非农。
  • 重大监管和政策。

#### 第 2 步：判断是否入选

逐条问：

```text

它是否影响油价、美元、美债、VIX 或全球风险偏好？

它是否影响美股、港股、A/H 科技或高估值资产？

它是否有明确来源？

它是否比普通新闻更重要？

```

四个问题至少三个为“是”，才能入选。

#### 第 3 步：每条只写三段

固定格式：

```text

事件：...

判断：...

来源：...｜置信度：...

```

不得新增第四段。

#### 第 4 步：写 `事件：`

规则：

  • 只写事实。
  • 一句话。
  • 30-60 个汉字为宜。
  • 不写影响。
  • 不写判断。

模板：

```text

事件：{主体}称/披露/公布/警告/确认{事实}。

```

例：

```text

事件：伊朗警告美国若再军事行动，将面临进攻性回应。

```

#### 第 5 步：写 `判断：`

规则：

  • 第一句写投资影响。
  • 第二句可写反向变量。
  • 不超过 2 句。

模板：

```text

判断：{事件}会{推升/压制/改善/削弱}{油价/风险偏好/科技股/港股流动性}。若{反向变量}，{判断调整}。

```

例：

```text

判断：美伊风险升级会推升油价和避险需求，压制科技股风险偏好。若谈判恢复且油价回落，冲击减弱。

```

#### 第 6 步：写 `来源：`

规则：

  • 至少 1 个主流财经或官方来源。
  • 快讯源 + 主流财经，可以高/中置信。
  • 只有快讯源，最多中置信。
  • 只有社媒或传闻，不得入正文。

模板：

```text

来源：金十数据、Xinhua｜置信度：高

```

#### 第 7 步：发布前搜索禁用标签

宏观大事件板块内不得出现：

```text

事件说明：

市场判断：

影响范围：

风险提示：

后续变量：

结论：

依据：

观察：

```

出现任一项，必须重写。

#### 第 8 步：数量检查

正常交易日或周末试运行版，宏观大事件不应只有 1 条。

如果只有 1 条：

  • 不能发布投资级版本。
  • 必须继续补源。
  • 至少检查金十、财联社、Reuters/AP/CNBC/Bloomberg、官方来源。

6. `3. 产业热点与科技主线` SOP

6.1 板块定位

第 3 部分不再以 IPO / 融资为主轴，而是以产业主线为主轴。

核心目标：

  • 识别 AI、半导体、算力、存储、云、机器人等产业主线。
  • 捕捉中美欧日韩产业链事件。
  • 解释事件如何映射到美股、港股、A/H 科技资产。
  • 辅助判断热点是否值得跟踪。

6.2 固定子模块

保留：

1. 今日产业主线判断。

2. AI / 半导体 / 算力链。

3. 重点公司财报 / 指引 / 订单 / 资本开支。

4. IPO / 融资 / 资本市场事件。

不得恢复独立 ETF / 基金产品空模块。

6.3 今日产业主线判断

只保留一段综合判断。

合格示例：

```text

DRAM 涨价验证 AI 存储链景气，A/H 半导体与美股存储链可跟踪；风险是 IPO 进度与存储周期反转。

```

不得重复具体条目的完整事件、判断、风险、后续变量。

6.4 产业热点条目格式

产业热点条目也使用三段式：

```text

事件：...

判断：...

来源：...｜置信度：...

```

合格示例：

```text

长鑫科技科创板 IPO 申报稿披露一季度业绩高增

事件：长鑫科技申报稿披露一季度营收同比增 719.13%，净利润约 330.12 亿元。

判断：DRAM 涨价验证 AI 存储链景气，半导体链情绪偏正面。若存储价格回落或审核不及预期。

来源：财联社、CFi.CN、CNR｜置信度：高

```

6.5 傻瓜操作步骤

写 `3. 产业热点与科技主线` 时按下面步骤执行。

#### 第 1 步：先判断今天有没有产业主线

必须检查：

```text

AI

算力

云 / AI Capex

机器人

自动驾驶

半导体设备
```

```text

```

不得强行填充低质量项目。

#### 第 2 步：写今日产业主线判断

只写一段。

模板：

```text

{产业变量}验证{主线景气/压力}，{相关资产}可跟踪；风险是{反向变量}。

```

例：


不得写三段：

```text

主线判断：

关键风险：

```

#### 第 3 步：写具体产业条目

格式固定：


不得出现：

#### 第 4 步：处理空子模块

如果 `重点公司财报 / 指引 / 订单 / 资本开支` 没有合格内容，写：

```text

```

如果 `IPO / 融资 / 资本市场事件` 只有作为产业主线的一部分，写：

```text
已在上方产业主线中按实际影响呈现；IPO / 融资仅作为事件属性，不再单独占据主轴。

```

不得恢复 ETF / 基金产品空模块。

7. 信源 SOP

7.1 宏观和产业热点必须全球核查

产业热点和宏观大事件不得只局限于美股信源。

必须覆盖：

  • 中国：财联社、金十数据、证券时报、上交所、深交所、HKEX、中国证监会。
  • 美国：Reuters、AP、CNBC、Bloomberg、SEC EDGAR、Nasdaq、NYSE、公司 IR。
  • 欧洲：Financial Times、Euronext、ASML IR。
  • 日本：Nikkei Asia、JPX/TSE、Tokyo Electron IR。
  • 韩国：Yonhap、Korea Exchange、Samsung Electronics IR、SK hynix IR。

7.2 快讯源使用规则

金十数据、财联社电报等快讯源可以作为线索触发源。

但：

  • 不得单独支撑高置信结论。
  • 必须尽量回查官方、公司 IR、交易所、SEC 或主流财经媒体。
  • 如果只有快讯源，置信度必须降级，并写明待确认变量。

7.3 信息源不足时不得发布投资级结论

如果宏观大事件只有 1 条，或者明显漏掉高影响事件：

  • 不得写“主线覆盖改善”。
  • 必须继续补源。
  • 质量检查应提示覆盖不足。

7.4 傻瓜补源步骤

当发现信息源不足时，按下面顺序补。

#### 第 1 步：快讯源扫一遍

检查：

```text

金十数据重要事件

财联社电报

```

目标：发现当天高影响线索。

注意：快讯源只负责发现，不负责最终定论。

#### 第 2 步：国际主流源确认

对每条宏观和产业线索，至少查一个：

```text

Reuters

AP

CNBC

Bloomberg

Financial Times

Nikkei Asia

Yonhap

```

如果查不到，不得直接删除线索；应降置信度或写待确认变量。

#### 第 3 步：官方源回填

按事件类型查：

```text

宏观：Fed、Treasury、BLS、BEA、CBOE

监管：SEC、FTC、DOJ、FDA、交易所

产业：公司 IR、交易所公告、招股书、财报

港股：HKEX、券商打新工具、暗盘数据

```

#### 第 4 步：确认置信度

置信度按以下规则：

```text

官方/公司 IR + 主流财经 = 高

主流财经交叉确认 = 高或中

快讯 + 主流财经 = 中或高，视事实清晰度决定

只有快讯 = 中或低

只有社媒/传闻 = 不入正文

```

#### 第 5 步：记录缺口

如果仍缺源，必须在内部日志或质量结果中记录：

```text

缺什么源

查过什么源

下一步查什么

是否影响发布

```

不得把“查不到”写成正文长段。

8. 版面密度 SOP

8.1 整体密度目标

页面要像投资简报，不像长文章。

要求：

  • 段落紧凑。
  • 条目之间区分清楚但不能拉太开。
  • 决策摘要卡片不能过高。
  • h2/h3 上下间距不能像文章标题。

8.2 推荐样式参数

港美股日报私密页推荐：

```css

.article-body { line-height: 1.55; }

.article-body h2 { margin-top: 30px; margin-bottom: 12px; }

.article-body h3 { margin-top: 18px; margin-bottom: 6px; }

.article-body p { margin-bottom: 4px; line-height: 1.55; }

.decision-cards { gap: 8px; margin: 12px 0 20px; }

.decision-card { padding: 12px 16px; line-height: 1.55; }

```

不得回退到：

  • 正文 `line-height: 1.9`。
  • h2 顶部 `48px`。
  • h3 顶部 `32px`。
  • 决策卡片 `line-height: 1.85`。

8.3 数据说明

数据说明只能作为弱化信息出现。

要求：

  • 放在板块底部。
  • 字号小。
  • 颜色弱。
  • 不占主视觉。

不得放在板块开头。

8.4 傻瓜版面检查步骤

生成 HTML 后，按肉眼检查和字符串检查双重执行。

#### 第 1 步：检查决策摘要

肉眼标准：

  • 三张卡片高度接近。
  • 每张卡片不超过两行半。
  • 卡片之间没有大块空白。

如果卡片看起来像正文大段落，FAIL。

#### 第 2 步：检查事件条目

每个宏观大事件条目应呈现为：

```text

标题

事件

判断

来源

```

标题和下一条标题之间应紧凑，但能分辨不同事件。

#### 第 3 步：检查 CSS 参数

HTML 中必须出现或等效覆盖：

```text

.article-body{line-height:1.55}

.article-body h2{margin-top:30px;margin-bottom:12px

.article-body h3{margin-top:18px;margin-bottom:6px

.article-body p{margin-bottom:4px

.decision-cards{gap:8px;margin:12px 0 20px}

.decision-card{padding:12px 16px
```

如果页面仍使用全站默认：

```text

line-height:1.9

h2 margin-top:48px

h3 margin-top:32px

decision-card line-height:1.85

```

必须修复后再发布。

10.7 发布失败处理

  • 不得修改无关文件绕过失败。

常见情况：

```text

git push 失败：说明网络或权限问题。

raw 主分支也失败：不得说已上线。

```

11. 自动化测试建议

应补充或保留以下断言：

  • HTML 包含 `<h2>2. 宏观大事件</h2>`。
  • HTML 不包含 `<h2>2. 美股重大事件</h2>`。
  • 宏观大事件条目包含 `事件：`、`判断：`、`来源：`。
  • 宏观大事件条目不包含 `事件说明：`、`市场判断：`、`影响范围：`、`风险提示：`、`后续变量：`。
  • 摘要区不包含港股打新项目名。
  • 摘要区不包含 `新股和融资机会不强行升级`。
  • 首页不包含 `ETF / 基金产品` 空模块。
  • 港股打新首页不包含完整项目表。
  • 页面内联样式覆盖简报密度参数。
  • 发布链接必须包含 `?v=<commit>`。
  • 发布目录只允许目标 HTML 发生变化。
